188. 买卖股票 IV

188. 买卖股票的最佳时机 IV

买卖股票有六种题型，后续更新

状态图：

由题意分析，不难得出：

  • 如果状态是是 买入 :

    • 那么可以 保持不动
    • 或者马上 卖掉

  • 如果状态是 卖出:

    • 可以 保持不动 (等待更好的股价出现，暂时不买)
    • 或者 立马再买一股
    • 同时将count数+1，表示交易过一次了

暴力递归思考问题

我们可以用 三个变量 来表示买卖k次的交易状态

  • index，用来表示 当前是哪一天
  • status，用来表示 当前是买入、还是卖出
  • count，用来表示 交易了几次

暴力法中我们尝试所有的解决策略：

    public int maxProfit1(int k, int[] prices) {
    // 暴力
    if(prices.length== 0 || prices == null){
        return 0;
    }
    return dfs(0, 0, 0, k, prices);
}

public int dfs(int index, int status, int count, int k, int[] prices){
    if (count == k || index == prices.length) {
        return 0;
    }
    // 保持不变
    int a = 0, b = 0, c = 0;
    a = dfs(index + 1, status, count, k, prices);
    if (status == 1){
        // 卖出
        b = dfs(index + 1, 0, count + 1, k, prices) + prices[index];
    }else {
        // 买入
        c = dfs(index + 1, 1, count, k, prices) - prices[index];
    }
    return Math.max(Math.max(a, b), c);
}

动态规划

三个参数三维表，但有一维为01，转为两个二维表

我们再次回顾一下状态转化：

每次到达i位置时：

如果三维表最后一位 0表示卖出，1表示买入：

第一次买入：从初始状态转换而来，或者第一次买入后保持不动
#dp[i][0][1] = max(dp[i-1][0][1],dp[i-1][0][0]-prices[i])

第一次卖出：从第一次买入转换而来，或者第一次卖出后保持不动
#dp[i][1][0] = max(dp[i-1][1][0],dp[i-1][0][1]+prices[i])

第二次买入：从第一次卖出转换而来，或者第二次买入后保持不动
dp[i][1][1] = max(dp[i-1][1][1],dp[i-1][1][0]-prices[i])

第二次卖出：从第二次买入转换而来，或者第二次卖出后保持不动
dp[i][2][0] = max(dp[i-1][2][0],dp[i-1][1][1]+prices[i])

第三次买入：
dp[i][2][1] = max(dp[i-1][2][1],dp[i-1][2][0]-prices[i])

第三次卖出：   
dp[i][3][0] = max(dp[i-1][3][0],dp[i-1][2][1]+prices[i])

扩展至j次交易的话：

第k次买入： 从第k-1次卖出转换而来，或者第k次买入后保持不动
dp[i][j-1][1] = max(dp[i-1][j-1][1],dp[i-1][j-1][0]-prices[i])

第k次卖出： 从第k次买入后转换而来，或者是第k次卖出后保持不动
dp[i][j][0] = max(dp[i-1][j][0],dp[i-1][j-1][1]+prices[i])

第k+1次买入：
dp[i][j][1] = max(dp[i-1][j][1],dp[i-1][j][0]-prices[i])

因此我们设置 两张表：

buy和sell，都为二维表，这样就可以忽略三维表第三位：

我们的规则如下：

buy[i] [j]表示到达索引i时候，达成j次交易买入获得最大收益
sell[i] [j]表示到达索引i时候，达成j次交易卖出获得最大收益
buy[i] [j] = Math.max( buy[i - 1] [j], sell[i - 1] [j] - price[i])
sell[i] [j] = Math.max( sell[i - 1] [j], buy[i - 1] [j - 1] + price[i])

    // 动态规划
public int maxProfit2(int k, int[] prices) {
    if(prices.length== 0 || prices == null){
        return 0;
    }
    int n = prices.length;
    k = Math.min(n/2, k);
    // 动态规划的意义
    // buy[i][j]表示到达索引i时候，达成j次交易买入获得最大收益
    // sell[i][j]表示到达索引i时候，达成j次交易卖出获得最大收益
    // buy[i][j] = Math.max( buy[i - 1][j], sell[i - 1][j] - price[i])
    // sell[i][j] = Math.max( sell[i - 1][j], buy[i - 1][j - 1] + price[i])
    int[][] buy = new int[n][k + 1];
    int[][] sell = new int[n][k + 1];
    buy[0][0] = -prices[0];
    sell[0][0] = 0;

    // 注意：buy[0][i] 和 sell[0][i] 是没有意义的
    for (int i = 1; i <= k; ++i) {
        buy[0][i] = sell[0][i] = Integer.MIN_VALUE / 2;
    }

    for (int i = 1; i < n; i++){
        buy[i][0] = Math.max( buy[i - 1][0], sell[i - 1][0] - prices[i]);
        for (int j = 1; j<= k; j++){
            buy[i][j] = Math.max( buy[i - 1][j], sell[i - 1][j] - prices[i]);
            sell[i][j] = Math.max( sell[i - 1][j], buy[i - 1][j - 1] + prices[i]);
        }
    }

    return Arrays.stream(sell[n - 1]).max().getAsInt();
}

动态规划优化

空间优化

上述中：

  • 外层的循环是n，即遍历数组
  • 内层的循环是k，遍历k次交易

不难看出上述二维表中的i并没有被实际用到

又或者说：状态转移方程中，buy[i] [j] 和 sell[i] [j] 都从 buy[i−1] [..] 以及 sell[i−1] [..] 转移而来

    public int maxProfit(int k, int[] prices) {
    // 动态规划空间优化
    if(prices.length== 0 || prices == null){
        return 0;
    }
    int n = prices.length;
    k = Math.min(n/2, k);

    int[] buy = new int[k + 1];
    int[] sell = new int[k + 1];
    buy[0] = -prices[0];
    sell[0] = 0;

    // 注意：buy[0][i] 和 sell[0][i] 是没有意义的
    for (int i = 1; i <= k; ++i) {
        buy[i] = sell[i] = Integer.MIN_VALUE / 2;
    }

    for (int i = 1; i < n; i++){
        buy[0] = Math.max( buy[0], sell[0] - prices[i]);
        for (int j = 1; j<= k; j++){
            buy[j] = Math.max( buy[j], sell[j] - prices[i]);
            sell[j] = Math.max( sell[j], buy[j - 1] + prices[i]);
        }
    }

    return Arrays.stream(sell).max().getAsInt();
}

buy[0][i] 和 sell[0][i] 是没有意义的,必须值为负最小

贪心算法

330. 按要求补齐数组

330. 按要求补齐数组

贪心策略

每次找到未被 数组nums 覆盖的 最小的整数x，在数组中 补充x，然后寻找下一个 未被覆盖的最小的整数 ，重复上述步骤直到区间 [1,n] 中的所有数字都被覆盖。

贪心原理及证明

对于正整数 x，如果区间 [1,x-1] 内的所有数字都已经被覆盖，且 x 在数组中，则区间 [1,2x-1] 内的所有数字也都被覆盖。证明如下。

对于任意 1≤y<x，y 已经被覆盖，x 在数组中，因此 y+x 也被覆盖，区间 [x+1,2x-1]（即区间 [1,x-1] 内的每个数字加上 x 之后得到的区间）内的所有数字也被覆盖，由此可得区间 [1,2x-1] 内的所有数字都被覆盖。

直接看证明可能有点 晦涩难懂 ，我们看一下 例子 ：

对于数组[1, 5, 10] 要求加x个数字能够覆盖20，即[1, 20]。

1⃣️ 我们设 初始能够覆盖的边界 为boundary = 1，即当前我们还没开始遍历数组，我们的 初始覆盖范围为[1, 1)

2⃣️ 现在我们开始遍历数组，从第一位开始，遍历到 1，当前数组自身第一个数就能够覆盖的范围为 [1, 1]，1<= boundary，所以我们加上boundary，覆盖范围变成 [1, 1 + boundary) ，即[1, 2)，注意右边为 开区间 ，你可以把boundary想象成 闭区间且初始没有覆盖为0 ；

3⃣️ 我们再遍历下一位，来到5，此时我们发现 [1, 2) 无法遍历到 5，于是我们加入 最小的整数x ，寻找新覆盖范围，能否覆盖到5，我们加入的数正根据上述的 贪心原理及证明 规则中加入， 边界[1, 2)我们加入2，所以新的覆盖范围为[1, 4)，加入数加一，我们再次来到5，发现还是无法遍历到5，我们再加入最小整数x，新的覆盖范围为[1, 8)，加入数加一。

4⃣️ 我们又一次来到5，发现可以覆盖到5了，但数组中的5改怎么办呢？

即因为数组中5存在，所以我们需要扩大我们的覆盖范围，实际上就是覆盖范围加上5，变为 [1, 8 + 5) ，证明上述已经证明，下一次来到10，发现可以覆盖10，又因为数组中有10，所以我们再次更新覆盖范围 [1, 8 + 5 + 10) ，最后覆盖范围能够越过20.

    public int minPatches(int[] nums, int n) {

    int len = nums.length;

    // 从1开始验证覆盖范围， 此时可覆盖的范围是[1,1)
    // boundary表示连续覆盖范围的右边界
    long boundary = 1; 

    // count 表示 添加了几个数字
    // i表示目前遍历到的nums中的index
    int index = 0;
    int count = 0; 

    // 若目前还没连续覆盖到n，则继续循环
    while (boundary <= n){
        // 原本覆盖范围为[1,add)，如果当前加入nums[i]到[i,boundary)中
        // 那么此时可构成的连续覆盖范围就是[1,boundary + nums[i])
        if ( (index < len ) && ( nums[index] <= boundary )){
            boundary += nums[index];
            index++;
        } else {
            // 否则，那么目前由数组中原本的数字没法继续扩大为下一级别的连续范围，
            // 则向数组中加入此时的右边界add，可覆盖[1,2boundary)范围  
            boundary = boundary << 1;
            count++;
        }
    }
    return count;

}

boundary应该小于等于n，该程序中边界为开区间
